# AI Output Review

## Review Summary

The AI helped generate structured drafts for inception, elicitation, requirements, prioritization, and validation. However, human review was necessary to remove ambiguity, label assumptions, improve traceability, and ensure requirements were testable.

## Skill 1: Project Inception and Stakeholder Discovery

### AI Output File

`outputs/raw/inception-ai-output.md`

### Problems Found

1. Some assumptions were written too generally.
2. Some scope items needed clearer in-scope and out-of-scope separation.

### Student Corrections

1. Added numbered assumptions.
2. Separated scope into in-scope and out-of-scope.
3. Added open questions about file type, resubmission, and late submission policy.

### Final Output

`outputs/reviewed/01-inception.md`

## Skill 2: Requirements Elicitation

### AI Output File

`outputs/raw/elicitation-ai-output.md`

### Problems Found

1. The raw output did not fully separate explicit and implicit needs.
2. Some findings needed clearer stakeholder mapping.

### Student Corrections

1. Added elicitation source IDs.
2. Added stakeholder-specific interview questions.
3. Separated explicit needs, assumptions, and open questions.

### Final Output

`outputs/reviewed/02-elicitation.md`

## Skill 3: Requirements Elaboration and Specification

### AI Output File

`outputs/raw/requirements-ai-output.md`

### Problems Found

1. AI used the vague term "quickly".
2. Some requirements did not include source IDs.
3. Non-functional requirements were not all measurable.

### Student Corrections

1. Replaced "quickly" with a measurable 3-second response time.
2. Added source references to all functional requirements.
3. Added measurable security, availability, and data integrity requirements.
4. Added business rules.

### Final Output

- `outputs/reviewed/03-requirements.md`
- `outputs/reviewed/04-user-stories.md`

## Skill 4: Negotiation and Prioritization

### AI Output File

`outputs/raw/prioritization-ai-output.md`

### Problems Found

1. Some prioritization reasons were too short.
2. Dependencies were missing from the first draft.
3. Stakeholder conflicts were not clearly explained.

### Student Corrections

1. Added detailed rationale for each priority.
2. Added dependency table.
3. Added stakeholder conflict and trade-off decisions.

### Final Output

`outputs/reviewed/05-prioritization.md`

## Skill 5: Requirements Validation and Change Management

### AI Output File

`outputs/raw/validation-ai-output.md`

### Problems Found

1. The first validation table was too brief.
2. Change impact was not detailed enough.

### Student Corrections

1. Added validation criteria for clarity, completeness, consistency, feasibility, testability, and traceability.
2. Added specific validation issues and corrections.
3. Expanded change request impact analysis.

### Final Output

- `outputs/reviewed/07-validation.md`
- `outputs/reviewed/08-change-request.md`
