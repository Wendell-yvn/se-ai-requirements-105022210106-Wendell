# Student Task Management System Case

## Case Description

Student Task Management System is a campus application used by lecturers, students, and administrators to manage academic assignments. The system supports assignment creation, deadline management, file submission, grading, feedback, user management, course management, and system configuration.

## Business Problem

Assignment management in a campus environment can become difficult when tasks, deadlines, submissions, grades, and feedback are handled across separate channels. Lecturers may find it hard to monitor submissions, students may miss deadlines, and administrators may struggle to maintain consistent course and user data.

## Project Goals

- Help lecturers create, manage, grade, and give feedback on assignments.
- Help students view assignments, submit files, and track assignment status.
- Help administrators manage users, courses, and system configuration.
- Improve assignment visibility, data integrity, usability, security, performance, and reliability.

## Stakeholders

- Lecturer
- Student
- Administrator
- Academic department
- IT support team

## Initial Scope

### In Scope

- Assignment creation and deadline management
- Student assignment viewing and file submission
- Submission status tracking
- Grade and feedback management
- User and course management
- Basic reporting
- Authentication and role-based access

### Out of Scope

- Online video lectures
- Real-time chat
- Automatic plagiarism detection
- Payment or finance features
- Full learning management system replacement

## Constraints

- The system must protect student academic data.
- The system must maintain submission data integrity.
- Requirements must be testable and traceable.
- Any information not provided in the case must be labeled as an assumption.

## Assumptions

- ASSUMPTION: The system will be accessed through a web application.
- ASSUMPTION: Users must log in before using role-specific features.
- ASSUMPTION: The university already has student, lecturer, and course data.
- ASSUMPTION: Uploaded assignment files must be stored securely.

## Open Questions

- OPEN QUESTION: What file types and maximum file sizes are allowed for assignment submission?
- OPEN QUESTION: Should students be allowed to resubmit before the deadline?
- OPEN QUESTION: Should late submissions be blocked or marked as late?
- OPEN QUESTION: What reports are required by administrators or academic departments?
